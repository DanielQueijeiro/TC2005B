Preguntas a responder

Laboratorio 1

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

    Internet es un sistema de computadoras que conforma una gran red de comunicación mundial, mientras que la World Wide Web es un sistema que funciona a través de Internet y permite transmitir diversos tipos de datos a través del Protocolo de Transferencia de Hipertextos (HTTP)

  2. ¿Cuáles son las partes de un URL?

    Protocolo: Especifica la forma en que se accederá al recurso.

    Subdominio: Todo lo que aparece entre el protocolo y el primer punto que lo separa del dominio.

    Dominio: La dirección del sitio web en sí.

    TLD (Top Level Domain): Es la extensión del dominio.

    Ruta: Es la ubicación precisa del recurso dentro del servidor y puede incluir carpetas y subcarpetas

  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    GET: Solicita una representación de un recurso específico. Las peticiones que usan el método GET sólo deben recuperar datos

    HEAD: Pide una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta

    POST: Se utiliza para enviar una entidad a un recurso en específico, causando a menudo un cambio en el estado o efectos secundarios en el servidor

    PUT: Reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición

    PATCH: Es utilizado para aplicar modificaciones parciales a un recurso

    DELETE: Borra un recurso en específico

  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    El método POST se utiliza para enviar un formulario HTML. Esto se debe a que el método POST permite enviar datos en el cuerpo de la solicitud, lo que es más seguro y privado que enviarlos en la URL, como lo hace el método GET

  5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    El método GET se utiliza cuando a través de un navegador web se accede a una página a través de un URL

  6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    El código de respuesta de estado satisfactorio HTTP 200 OK indica que la solicitud ha tenido éxito

  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    Sí, es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404. Esto se debe a que el error 404 indica que el recurso solicitado no se encuentra en el servidor, lo que podría ser el resultado de un enlace roto o de un recurso que ha sido eliminado.

  8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    Sí, es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500. Este error indica un problema en el servidor que impide procesar la solicitud.

  9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    Cuando un atributo HTML5 está depreciado o desaprobado, significa que ya no se recomienda su uso porque ha sido reemplazado por una opción más nueva o mejor.

    Algunos ejemplos de elementos desaprobados son: Applet, acronym, bgsound, dir, hgroup

  10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    HTML5 es la versión más reciente y tiene varias mejoras y adiciones en comparación con HTML 4.

    HTML5 ofrece una sintaxis más flexible y coherente en comparación con HTML4.

    HTML5 introduce elementos como video y audio para un manejo más fácil de contenido multimedia.

    Nuevos elementos semánticos: HTML5 añade elementos semánticos como article, section, nav, header, footer, que ayudan a mejorar la estructura y la interpretación del contenido de la página web

  14. ¿Cuál es el ciclo de vida de los sistemas de información?

    El ciclo de vida de un sistema de información es un modelo estructurado de gestión de proyectos que describe las fases necesarias para construir un sistema de información, desde el inicio hasta el resultado final. Este ciclo se compone de varias etapas:

    Planificación: Se decide lo que el sistema pretende lograr.

    Análisis de requerimientos: Se comprende qué se necesita para desarrollar con éxito el sistema.

    Diseño: Se estudian posibles alternativas de implementación y se decide la estructura general que tendrá el sistema.

    Desarrollo: Se seleccionan las herramientas adecuadas, un entorno de desarrollo que facilite nuestro trabajo y un lenguaje de programación apropiado para el tipo de sistema que vayamos a construir.

    Pruebas: Esta fase tiene como objetivo detectar los errores que se hayan podido cometer en las etapas anteriores del proyecto (y, eventualmente, corregirlos).

    Implementación: Se planifica el entorno en el que el sistema debe funcionar, tanto hardware como software.

    Operación y mantenimiento: Esta etapa consume típicamente del 40 al 80 por ciento de los recursos de una empresa de desarrollo de software. Se eliminan los defectos que se detecten durante su vida útil y se adapta el sistema a nuevas necesidades.

  15. ¿Cuál es el ciclo de desarrollo de sistemas de información?

    El ciclo de desarrollo de sistemas de información, también conocido como ciclo de vida del desarrollo de software (SDLC), es un proceso estructurado que los equipos de desarrollo utilizan para diseñar y crear software de alta calidad Este ciclo se compone de varias etapas:

    Planificación: Se recopilan los requisitos de varias partes interesadas para crear un documento de especificaciones con los requisitos del software.

    Análisis de requerimientos: Se comprende qué se necesita para desarrollar con éxito el sistema.

    Diseño: Se analizan los requisitos e identifican las mejores soluciones para crear el software.

    Desarrollo: Se seleccionan las herramientas adecuadas, un entorno de desarrollo que facilite nuestro trabajo y un lenguaje de programación apropiado para el tipo de sistema que vayamos a construir.

    Pruebas: Esta fase tiene como objetivo detectar los errores que se hayan podido cometer en las etapas anteriores del proyecto (y, eventualmente, corregirlos).

    Implementación: Se planifica el entorno en el que el sistema debe funcionar, tanto hardware como software.

    Operación y mantenimiento: Esta etapa consume típicamente del 40 al 80 por ciento de los recursos de una empresa de desarrollo de software. Se eliminan los defectos que se detecten durante su vida útil y se adapta el sistema a nuevas necesidades.

  16. ¿Cuál es la diferencia entre el ciclo de vida y el ciclo de desarrollo del software?

    El ciclo de vida del software y el ciclo de desarrollo del software son dos conceptos que a menudo se utilizan de manera intercambiable, pero tienen matices que los diferencian:

    El ciclo de vida del software se refiere a la secuencia completa de actividades o fases que un software atraviesa desde que se concibe hasta que se retira o deja de usarse. Este ciclo incluye fases como la concepción, el desarrollo, la operación, el mantenimiento y, finalmente, la retirada del software.

    Por otro lado, el ciclo de desarrollo del software, también conocido como ciclo de vida del desarrollo de software (SDLC), se centra más en el proceso de desarrollo del software. Este ciclo incluye fases como la planificación, el análisis de requisitos, el diseño, la codificación, las pruebas, la implementación y el mantenimiento.

    Cada unidad tiene sus ventajas y desventajas, lo importante es saber que unidad es la adecuada usar para que sea eficiente y tenga encuenta la accesibilidad y portabilidad.

Laboratorio 4

  1. ¿Qué diferencias y semejanzas hay entre Java y JavaScript?
    Java JavaScript
    Java es un lenguaje compilado JavaScript es un lenguaje interpretado
    Java es un lenguaje de programación orientado a objetos JavaScript está basado en prototipos y puede emular la programación orientada a objetos
    JavaScript se ejecuta en el navegador del usuario final Java requiere un entorno de ejecución llamado Java Virtual Machine (JVM) para ejecutar el código.
    JavaScript es débilmente tipado Java es fuertemente tipado
  2. ¿Qué métodos tiene el objeto Date?

    El método getDate() devuelve el día del mes para la fecha especificada de acuerdo con la hora local.

    El método getFullYear() devuelve el año de la fecha indicada acorde a la hora local.

    El método getMinutes() devuelve los minutos de la fecha especificada en función de la hora local.

    El método getTime() devuelve el valor numérico correspondiente a la hora para la fecha especificada según la hora universal.

    El método now devuelve los milisegundos transcurridos desde las 00:00:00 UTC del 1 de enero de 1970 justo hasta ahora como un Number.

  3. ¿Qué métodos tienen los arreglos?

    El método concat() se usa para unir dos o más arrays. Este método no cambia los arrays existentes, sino que devuelve un nuevo array.

    El método find() devuelve el valor del primer elemento del array que cumple la función de prueba proporcionada.

    El método push() añade uno o más elementos al final de un array y devuelve la nueva longitud del array.

    El método Array.of() crea una nueva instancia Array con un número variable de elementos pasados como argumento, independientemente del número o del tipo.

    El método reduce() ejecuta una función reductora sobre cada elemento de un array, devolviendo como resultado un único valor.

  4. ¿Cómo se declara una variable con alcance local dentro de una función?

    Con la declaración "let" la variable será local con ámbito de bloque, opcionalmente la inicia a un valor.

  5. ¿Qué implicaciones tiene utilizar variables globales dentro de funciones?

    Problemas de mantenibilidad y colisiones de nombres: El uso excesivo de variables y funciones globales puede llevar a problemas de mantenibilidad y colisiones de nombres. Esto puede hacer que el código sea más difícil de entender y mantener.

    Modificación de variables globales: Una variable global puede ser utilizada e incluso modificada dentro de una función. Esto puede llevar a comportamientos inesperados si no se maneja correctamente.
